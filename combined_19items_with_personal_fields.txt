# Gabungan 19 Item Tambahan dengan Data Pribadi / Form Fields
Dokumentasi ini menggabungkan **19 item tambahan** (data perangkat, network, sensor, perilaku, dsb.) dengan **data pribadi / field form** yang kemungkinan terkait dan dikirim ke server oleh aplikasi (berdasarkan analisis statik APK).

> Catatan: Ini adalah hasil analisis statik + inferensi. Untuk bukti 100% lakukan dekompilasi (jadx/apktool) dan network capture (mitmproxy/Burp).

---

## Cara baca dokumen
Setiap entri berisi:
- **Item** (data teknis/perangkat/behavioral)  
- **Data pribadi / form fields terkait** (field spesifik pada form atau data yang dapat dikorelasikan)  
- **Contoh nama parameter** (yang sering muncul di payload)  
- **Kapan / di mana dikirim** (step alur: register, apply, upload, background, dll.)  
- **Confidence** (Tinggi / Sedang / Rendah)  
- **Verifikasi singkat** (grep pattern / network filter / manifest)

---

### 1. Advertising ID (GAID / adid)\n\n- **Data pribadi / form fields terkait:** advertising_id / adid / gaid\n- **Contoh parameter / key:** advertising_id, adid, gaid, ad_id\n- **Kapan / di mana dikirim:** App launch / analytics registration; sent to analytics (Adjust) and backend mapping to user profile\n- **Confidence:** Sedang\n- **Verifikasi singkat:** grep AdvertisingIdClient; filter HAR for adid/advertising_id; check analytics payloads\n\n---\n\n### 2. IMSI / Subscriber ID / SIM Serial\n\n- **Data pribadi / form fields terkait:** phone / nomor_hp / sim_operator\n- **Contoh parameter / key:** imsi, subscriber_id, sim_serial, sim_operator\n- **Kapan / di mana dikirim:** Device registration / auth flows; attached to device metadata and telemetry\n- **Confidence:** Sedang\n- **Verifikasi singkat:** grep getSubscriberId|getSimSerialNumber; search HAR for subscriber_id keys\n\n---\n\n### 3. Android build & security info (root, app signature)\n\n- **Data pribadi / form fields terkait:** affects handling of access_token / session (not direct personal field)\n- **Contoh parameter / key:** is_rooted, signature, verified_boot\n- **Kapan / di mana dikirim:** App launch / device attestation; sent to backend for fraud/security checks\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** grep isRooted|PackageInfo.signatures; check payload for rooted flag\n\n---\n\n### 4. Device locale, timezone, language\n\n- **Data pribadi / form fields terkait:** affects display of dob / address formats; related to user locale fields\n- **Contoh parameter / key:** locale, language, timezone\n- **Kapan / di mana dikirim:** Any request (common in analytics & device registration)\n- **Confidence:** Sedang\n- **Verifikasi singkat:** search payloads for locale/timezone/lang\n\n---\n\n### 5. Public IP / network operator / network type\n\n- **Data pribadi / form fields terkait:** used to correlate location/city with address field\n- **Contoh parameter / key:** ip, network_operator, network_type\n- **Kapan / di mana dikirim:** Server receives automatically on requests; also in device registration/telemetry\n- **Confidence:** Sedang\n- **Verifikasi singkat:** inspect request headers (X-Forwarded-For) or HAR; grep network_operator in payloads\n\n---\n\n### 6. Wi‑Fi SSID / BSSID / MAC / Bluetooth MAC\n\n- **Data pribadi / form fields terkait:** can correlate with address or location fields\n- **Contoh parameter / key:** ssid, bssid, wifi_mac, bluetooth_mac\n- **Kapan / di mana dikirim:** Background telemetry / device fingerprinting; requires location/network permission\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** grep WifiManager.getConnectionInfo or capture HAR for ssid/bssid values\n\n---\n\n### 7. Installed apps list (package names)\n\n- **Data pribadi / form fields terkait:** profiling: may indicate other financial apps user has\n- **Contoh parameter / key:** installed_apps, packages, app_list\n- **Kapan / di mana dikirim:** Telemetry / anti-fraud endpoints during device registration or background sync\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** grep getInstalledPackages|getInstalledApplications; search HAR for app_list keys\n\n---\n\n### 8. Usage stats / app usage events\n\n- **Data pribadi / form fields terkait:** behavioral signals that can be tied to income/job patterns or app usage\n- **Contoh parameter / key:** usage_events, screen_time, last_opened\n- **Kapan / di mana dikirim:** Telemetry / analytics events (background or on app interactions)\n- **Confidence:** Rendah\n- **Verifikasi singkat:** grep UsageStatsManager; look for events arrays in telemetry payloads\n\n---\n\n### 9. Clipboard contents\n\n- **Data pribadi / form fields terkait:** may contain OTP codes, account numbers, copied personal data (name, bank)\n- **Contoh parameter / key:** clipboard_text, copied_text\n- **Kapan / di mana dikirim:** At moment of app open or during form fills if clipboard read occurs\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** grep ClipboardManager.getPrimaryClip; search HAR for copied text in payloads\n\n---\n\n### 10. Calendar / events\n\n- **Data pribadi / form fields terkait:** may reveal addresses or meeting info (personal)\n- **Contoh parameter / key:** calendar_events, event_title, event_location\n- **Kapan / di mana dikirim:** If app requests calendar permission; unlikely for loan app but possible\n- **Confidence:** Rendah\n- **Verifikasi singkat:** check READ_CALENDAR permission in manifest; grep calendar APIs\n\n---\n\n### 11. Files on storage (other documents)\n\n- **Data pribadi / form fields terkait:** scans or documents: additional KTP copies, bank statements, invoices\n- **Contoh parameter / key:** file (multipart), file_type, document_type, file_path\n- **Kapan / di mana dikirim:** Upload endpoints (multipart/form-data) during KYC or support flows\n- **Confidence:** Sedang\n- **Verifikasi singkat:** capture multipart POSTs in HAR; grep File/MediaStore accesses\n\n---\n\n### 12. Microphone recordings / ambient audio\n\n- **Data pribadi / form fields terkait:** could capture conversations mentioning personal data (bank details)\n- **Contoh parameter / key:** audio_file, recording\n- **Kapan / di mana dikirim:** If app has audio features or hidden recording; requires RECORD_AUDIO permission\n- **Confidence:** Rendah\n- **Verifikasi singkat:** grep AudioRecord/MediaRecorder and check RECORD_AUDIO permission\n\n---\n\n### 13. Camera metadata & EXIF (timestamp, GPS)\n\n- **Data pribadi / form fields terkait:** EXIF GPS coordinates can map to address; timestamp to activity time\n- **Contoh parameter / key:** (embedded EXIF in uploaded image) GPSLatitude, GPSLongitude, DateTime\n- **Kapan / di mana dikirim:** When uploading selfie/KTP images; stored in multipart file data\n- **Confidence:** Sedang\n- **Verifikasi singkat:** inspect uploaded image bytes for EXIF tags in HAR; check image processing code\n\n---\n\n### 14. SMS body / OTP / call logs\n\n- **Data pribadi / form fields terkait:** OTP codes, SMS content; call history can indicate contacts/behavior\n- **Contoh parameter / key:** sms_body, otp, call_logs\n- **Kapan / di mana dikirim:** Verification flows (OTP) or if app listens to SMS broadcasts\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** check manifest for RECEIVE_SMS/READ_SMS; capture verify-otp requests\n\n---\n\n### 15. Notification contents / read notifications\n\n- **Data pribadi / form fields terkait:** notifications may contain OTP or messages from other apps with personal info\n- **Contoh parameter / key:** notification_text, notification_title\n- **Kapan / di mana dikirim:** If app has NotificationListener or Accessibility services enabled\n- **Confidence:** Rendah\n- **Verifikasi singkat:** grep NotificationListenerService or BIND_ACCESSIBILITY_SERVICE\n\n---\n\n### 16. Event streams / clickstreams / screen interaction traces\n\n- **Data pribadi / form fields terkait:** user interactions that correlate with profile: income, job, loan behavior\n- **Contoh parameter / key:** events[], screen_name, action, timestamp\n- **Kapan / di mana dikirim:** Telemetry / analytics (Adjust) from UI interactions and flows\n- **Confidence:** Sedang\n- **Verifikasi singkat:** capture telemetry payloads for events arrays; grep for events in code\n\n---\n\n### 17. Crash traces & logs\n\n- **Data pribadi / form fields terkait:** may inadvertently include personal field values if logged (ktp, account_number)\n- **Contoh parameter / key:** stacktrace, error_message, log\n- **Kapan / di mana dikirim:** On crash or periodic error reporting to backend or third-party\n- **Confidence:** Sedang\n- **Verifikasi singkat:** grep for Sentry/Crashlytics integration; search for stacktrace upload endpoints\n\n---\n\n### 18. Hardware identifiers (serial, bluetooth address, sensor IDs)\n\n- **Data pribadi / form fields terkait:** additional device fingerprinting fields tied to user profile\n- **Contoh parameter / key:** device_serial, bt_mac, sensor_id\n- **Kapan / di mana dikirim:** Device registration / telemetry / anti-fraud endpoints\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** grep getSerial|getBluetoothAddress; search HAR for serial/bt_mac\n\n---\n\n### 19. Accessibility events / screen contents for OTP scraping\n\n- **Data pribadi / form fields terkait:** harvested screen text can include OTP, messages, personal forms\n- **Contoh parameter / key:** (no fixed key) harvested_text -> posted as sms_body/notification_text\n- **Kapan / di mana dikirim:** When Accessibility service is enabled and abused for auto-fill or scraping\n- **Confidence:** Rendah–Sedang\n- **Verifikasi singkat:** check manifest for BIND_ACCESSIBILITY_SERVICE; grep for classes extending AccessibilityService\n\n---\n\n
